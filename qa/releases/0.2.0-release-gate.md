## Decision

**GO WITH DOCUMENTED LIMITATIONS**

Safe for a controlled local live demonstration. No release blockers.

### Documented limitations

1. Operator machine must use **Node 22.x** for `demo:prepare`, `demo:start`, `demo:reset`, `demo:seed`, and `demo:smoke` CLI wrappers.
2. Two **moderate** npm audit findings (`uuid` via `exceljs`) — accepted at `npm audit --audit-level=high`.
3. pt-PT diacritics visual polish (FORGE-QA-001) — cosmetic, non-blocking.
4. Full deterministic restore / `seedDatabase(force=true)` wipes entire tenant scope (by design; distinct from demo-only reset).
5. Run `validate:release` sequentially after `validate:full` to avoid Next.js dev-lock contention on a single worktree.
