# ForgeOS 0.2.0 responsive audit

**Date:** 2026-07-05  
**Branch:** `release/forgeos-0.2.0-local-demo`  
**Method:** Playwright acceptance — horizontal overflow check on all critical routes

## Viewports tested

| Viewport | Width × Height | Result |
|----------|----------------|--------|
| Desktop large | 1920 × 1080 | Pass — no overflow > 40px on critical routes |
| Desktop | 1440 × 900 | Pass |
| Laptop | 1366 × 768 | Pass |
| Tablet landscape | 1024 × 768 | Pass |
| Tablet portrait | 768 × 1024 | Pass |
| Mobile | 390 × 844 | Pass |

**Threshold:** `document.documentElement.scrollWidth - clientWidth < 40` per route.

## Routes per viewport (12 each)

Login, dashboard, customers, LeadOps, campaigns, products, Cup Customizer, quotations, production, machines, inventory, settings.

## Layout defects found

| ID | Viewport | Issue |
|----|----------|-------|
| UI-0.2.0-R001 | 390 × 844 | Mobile nav drawer links blocked by main content (stacking context inside `overflow-x-clip` section) |

## Fixes applied

| ID | Fix |
|----|-----|
| UI-0.2.0-R001 | Portal drawer to `document.body` with `OVERLAY_Z_INDEX.mobileNavBackdrop` (40) and `OVERLAY_Z_INDEX.modal` (50) |

## Mobile-specific checks

| Drawer link navigates to `/pt-PT/customers` | Pass (after portal fix) |
