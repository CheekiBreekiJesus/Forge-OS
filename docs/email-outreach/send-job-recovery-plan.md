## Persistence Model

| Mode | Authority |
|------|-----------|
| Local simulation / tests | IndexedDB projections |
| Production real send jobs | Supabase tables via server-only helper |
| Public unsubscribe / webhooks | Existing Step 8 Supabase tables |

## ID Strategy

Supabase durable send-job IDs are UUIDs generated by the database. Local MVP entity IDs remain text references in columns such as `campaign_ref`, `campaign_recipient_ref`, and `lead_ref`. Future hosted sync should preserve that boundary instead of trying to cast local string IDs to UUIDs.
