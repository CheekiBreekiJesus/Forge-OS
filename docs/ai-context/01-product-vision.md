# 01 Product Vision

## 1. Long-Term Vision

ForgeOS should become a modular Industrial OS for small and medium manufacturing companies.

Long-term, it should support:
- Factory operations
- Warehouse management
- Maintenance
- Sales and CRM
- Production planning
- Technical documentation
- Website and marketing automation
- AI copilots and agent workflows
- Business intelligence
- Multi-tenant SaaS deployment

The platform should eventually help companies digitize operations without needing multiple disconnected tools.

## 2. Target Users

Initial target users:
- Business owners
- Production managers
- Machine operators
- Warehouse workers
- Sales/admin staff
- Marketing staff

First real-world tenant:
- JH Gomes

Future target industries:
- Plastic injection
- Mold manufacturing
- CNC machining
- Packaging
- Industrial assembly
- Automation integration

## 3. Main Use Cases

Initial use cases:
- Register customers
- Create and manage orders
- Create production jobs
- Generate technical production sheets
- Track inventory
- Track machine setup parameters
- Document troubleshooting and procedures
- Manage cup printing product configurations
- Support website and marketing content generation
- Generate internal newsletters
- Use AI assistance for administrative and operational tasks

## 4. Product Modules / Features

Planned modules:
1. Dashboard
2. CRM
3. Orders
4. Production
5. Inventory / Warehouse
6. Machines
7. Maintenance
8. Products
9. Technical Sheets
10. Marketing
11. Website / CMS
12. Internal Newsletter
13. AI Copilot
14. Analytics / BI
15. Admin / Tenant Settings

MVP modules should be a smaller subset.

## 5. Business Model

Assumption:
- Multi-tenant SaaS subscription model.
- Possible implementation/service revenue for industrial onboarding.
- Possible AI workflow/automation consulting revenue.

Not yet defined:
- Pricing tiers
- Billing provider
- Trial model
- Per-seat pricing
- Per-tenant pricing
- Usage-based AI pricing

## 6. Success Criteria

MVP success:
- JH Gomes can use ForgeOS for real daily operational workflows.
- Basic customer/order/product/job data is centralized.
- Production sheets reduce setup errors.
- Inventory information is easier to access than current manual methods.
- Portuguese UI is usable by factory/admin users.
- The app architecture supports additional tenants later.

Long-term success:
- Platform can be deployed to other manufacturing SMEs.
- New modules can be added without rewriting the core.
- AI features save measurable administrative time.
- Factory users adopt the system without heavy training.
