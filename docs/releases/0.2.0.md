# ForgeOS 0.2.0 — Local Demo Release

**Release type:** Local-demo release candidate  
**Branch:** `release/forgeos-0.2.0-local-demo`  
**Target integration branch:** `integration/jh-gomes-release-candidate`  
**Date:** 2026-07-05

## Scope

ForgeOS 0.2.0 delivers a stable, repeatable **browser-only local demo** for JH Gomes walkthroughs. All business data lives in IndexedDB; no hosted database mutations, real email, or paid AI calls occur in the demo contract.

### In scope

| Area | Coverage |
|------|----------|
| CRM | Customers, opportunities |
| LeadOps | Leads, campaigns, import, simulation send jobs |
| Commercial | Cup customizer, quotations, quote → production |
| Operations | Production orders, machines, inventory |
| Settings | Company profile, senders, backup/restore, about panel |
| Demo lifecycle | Deterministic seed, reset, JSON backup/restore |

### Out of scope

- Supabase-hosted persistence and RLS migrations
- Google / Microsoft OAuth
- Brevo and Outlook live send
- Abacus / OpenAI paid AI
- Maintenance module (placeholder)
- Sales orders, molds, quality preview shells

## Runtime requirements

| Item | Value |
|------|-------|
| Node.js | **22.x** (`engines`: `>=22 <23`) |
| npm | **10.9.8** (`packageManager` pin) |
| Persistence | `FORGEOS_PERSISTENCE_MODE=local` |
| Database | `forgeos:jhgomes:0.2.0-demo` |
| Schema version | **13** (`SCHEMA_VERSION`) |
| Seed version | **5** (`SEED_VERSION`) |

## Database schema compatibility

| From | To 0.2.0 | Action |
|------|----------|--------|
| 0.1.x dev DB (`forgeos:jhgomes:development`) | Not auto-migrated | Use demo scripts or dedicated demo DB name |
| 0.2.0 demo DB | Same build | Idempotent reseed when `SEED_VERSION` matches |
| JSON backup schema 13 | 0.2.0 | Compatible via Settings → import |
| JSON backup schema &lt; 13 | 0.2.0 | Import rejected (`isBackupSchemaCompatible`) |
| Hosted Supabase | N/A | Demo contract disables hosted mode |

Dexie schema version **13** is defined in `src/persistence/db.ts`. Bumping `SCHEMA_VERSION` triggers IndexedDB upgrade on next open.

## Upgrade

1. Check out `release/forgeos-0.2.0-local-demo`.
2. Use **Node 22.x** and `npm ci`.
3. Run `npm run demo:prepare` (or manual `npm ci`).
4. Start demo: `npm run demo:start`.
5. Reset browser data: `npm run demo:reset` (server must be running).
6. Verify: `npm run demo:smoke` or `npm run validate:release`.

Existing browser data in other IndexedDB names is untouched.

## Rollback

1. Check out previous tag or `integration/jh-gomes-release-candidate` at prior commit.
2. `npm ci` with Node 22.
3. Clear demo IndexedDB in browser DevTools **or** run `demo:reset` on the older build.
4. Restore from JSON backup exported before upgrade (Settings → export).

Manual git tag rollback (after approval):

```bash
git checkout v0.1.0   # or prior integration SHA
npm ci
npm run demo:start
```
